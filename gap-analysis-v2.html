The Infrastructure Advantage Is Gone

Our V1 analysis claimed 5 advantages rooted in Nedzo’s Bubble.io limitations: real codebase, performance, HIPAA eligibility, real whitelabel, and unlimited customization. Nedzo V2 eliminates all five. Both products are now custom-built React + API applications. The competition has shifted from “better tech stack + graph memory” to “graph memory alone.”

Honest Assessment

Advantages We Lost

Five advantages from our V1 analysis no longer hold against Nedzo V2. Acknowledging this is step one.

“Real Codebase”

Was: Strong differentiator

We claimed “real Python/TypeScript codebase vs no-code ceiling.” Nedzo V2 is now React + Vite + custom API. Both are real codebases.

NEUTRALIZED

Performance

Was: 55MB vs ~1MB

Nedzo V1 loaded 55MB of Bubble.io runtime per session. V2 uses a single Vite bundle (~1-5MB). Load times are now comparable.

NEUTRALIZED

HIPAA Eligibility

Was: Blocked for Nedzo

Bubble.io couldn’t sign a BAA. Custom backend at api.nedzo.ai can control data handling, encryption, and compliance. Healthcare vertical is now open to them.

NEUTRALIZED

Real Whitelabel

Was: CSS proxy vs real

V1 used CoAlias CSS proxy — browser dev tools revealed the underlying Bubble structure. V2’s custom codebase enables genuine multi-tenant whitelabeling.

NEUTRALIZED

Customization Ceiling

Was: No-code limits them

Bubble.io imposed hard limits on data models, performance, and customization. With a custom codebase, Nedzo can now build anything. Feature velocity should increase.

NEUTRALIZED

Faster Iteration Risk

New threat from V2

Custom codebase means Nedzo’s dev team can now ship features faster than on Bubble.io. The feature velocity gap may close. 400+ agencies being migrated creates retention lock-in.

NEW RISK

Still Ours

What Nedzo V2 Still Cannot Replicate

Structural advantages that survived the rewrite. These require Nedzo to rebuild their data layer — not just their frontend.

Primary Moat

Graph Memory That Compounds

Neo4j knowledge graph stores every interaction as connected nodes. Contacts, calls, entities, objections, buying signals, competitor mentions, relationship maps. Call #5 with a prospect automatically references calls #1–4. Nedzo V2’s React rewrite does nothing to address stateless calls. This is a data architecture gap, not a frontend gap.

Why V2 Doesn’t Fix This:

Rewriting from Bubble to React changes the UI layer. Graph memory requires a fundamentally different data model — a knowledge graph, entity extraction pipeline, and cross-session state management. You don’t get this by switching frameworks.

Compounds Over Time

Post-Call Intelligence Pipeline

Claude-powered entity extraction after every call: companies, people, products, objections, buying signals — all stored as graph nodes with relationships. Nedzo V2 still produces flat call summaries. The difference: structured, queryable intelligence vs. disposable text.

Provider Transparency + Local

6 TTS, 5 STT, 5+ LLM providers — user chooses. MLX Whisper, Chatterbox TTS, Ollama run fully on-device. $0.00/min floor.

Nedzo V2: still opaque, still cloud-only, still Vapi-dependent.

Developer-First API

132+ documented REST endpoints. OpenAPI auto-docs. Full programmatic control. Any integration partner or technical buyer chooses Vox.

Nedzo V2: ndz_ API keys exist but zero public documentation.

A/B Testing

Weighted prompt variants assigned at call start. Outcome tracking per variant. Systematic prompt optimization — no competitor has this.

Nedzo V2: no A/B testing.

Deep Analytics

Funnel visualization, cohort analysis, cost tracking, agent comparison, email reports (Postmark/SendGrid). Prometheus /metrics + Grafana ready.

Nedzo V2: basic end-of-call summaries.

Multi-Channel

Voice + SMS (two-way) + Email (Postmark/SendGrid, HTML, delivery tracking) + Unified inbox with unread counts and cross-channel timeline.

Nedzo V2: voice + web widget. No SMS, no email.
